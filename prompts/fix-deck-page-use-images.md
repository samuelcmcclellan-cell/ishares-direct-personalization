# Prompt: Fix Strategy Deck Page — Use Exported Slide Images Instead of HTML Recreation

## Context

The `/deck` route (`src/pages/DeckPage.jsx`) currently recreates both presentation slides as HTML/CSS components (`Slide1()` lines 10–48 and `Slide2()` lines 51–127). These look bad compared to the actual PowerPoint because web fonts, Tailwind spacing, and `clamp()` responsive sizing can't match the pixel-perfect layout of the real deck.

The actual PowerPoint file (`public/deck/iShares_Direct_Personalization_2_Slides.pptx`, generated by `build-deck.cjs`) looks great. And there are already exported JPG images of each slide sitting in `public/deck/`:

- `public/deck/slide-1.jpg` (181 KB) — Slide 1: "From ETF Provider to Personalization Engine"
- `public/deck/slide-2.jpg` (237 KB) — Slide 2: "Three Ways to Build This"

## Task

Rewrite `src/pages/DeckPage.jsx` to display the JPG images instead of the HTML slide recreations. The slide viewer chrome (top bar, navigation arrows, download button, slide toggle, slide counter, Framer Motion transitions) should stay exactly the same — only the slide content rendering changes.

### What to change

1. **Delete the `Slide1()` function** (lines 10–48) and the `Slide2()` function (lines 51–127). These are the HTML recreations that look bad.

2. **Delete the `MODEL_COLORS` and `AI_LABELS` constants** (lines 7–8) and the `OPERATING_MODELS` import (line 5). These were only used by the HTML slide components.

3. **Update the `SLIDES` array** (lines 129–132) to reference image paths instead of components:

```js
const SLIDES = [
  { label: 'The Opportunity', image: '/deck/slide-1.jpg' },
  { label: 'Three Ways to Build This', image: '/deck/slide-2.jpg' },
]
```

4. **Update the slide viewer** in the `DeckPage` component to render an `<img>` tag instead of a component. Replace the `<SlideComponent />` render (around line 210) with:

```jsx
<img
  src={SLIDES[active].image}
  alt={SLIDES[active].label}
  className="w-full rounded-lg shadow-2xl"
  draggable={false}
/>
```

5. **Remove the `SlideComponent` variable** (line 136: `const SlideComponent = SLIDES[active].component`) — it's no longer needed.

### What NOT to change

- The top bar with "Back to app" link, "iShares / Strategy Deck" branding, and "Download .pptx" button (lines 140–165) — keep exactly as-is
- The slide toggle buttons (lines 168–182) — keep exactly as-is
- The navigation arrows (lines 188–201) — keep exactly as-is
- The Framer Motion `<motion.div>` wrapper with fade/scale animation (lines 204–211) — keep the wrapper, just change what's inside it
- The slide counter text (lines 213–216) — keep exactly as-is
- The dark background styling and overall page layout — keep exactly as-is

## After editing

1. Start the dev server: `npm run dev`
2. Navigate to `http://localhost:5173/deck`
3. Verify:
   - Slide 1 image displays correctly, fills the viewer width, has rounded corners and shadow
   - Slide 2 image displays correctly
   - Arrow navigation and toggle buttons switch between slides with the Framer Motion animation
   - Download button still downloads the .pptx file
   - "Back to app" link still works
